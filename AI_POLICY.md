# AI-Assisted Contribution Policy

This project allows responsible use of AI tools, including large language models, code assistants, documentation assistants, and automated refactoring tools.

AI tools may help contributors draft, review, test, explain, or refactor code. However, AI-generated output is not accepted as a substitute for human understanding, review, authorship responsibility, or license compliance.

## Core Principle

Contributors are responsible for every contribution they submit, whether it was written manually, AI-assisted, generated by a tool, copied from an allowed source, or adapted from prior work.

By contributing to this project, you certify that you understand the contribution, have the right to submit it, and agree to maintain the same quality, security, and licensing standards expected of human-written code.

## Allowed Uses of AI

AI tools may be used for:

- drafting initial implementations;
- generating tests;
- explaining existing code;
- improving documentation;
- suggesting refactors;
- writing examples;
- producing boilerplate;
- checking edge cases;
- reviewing code for readability, bugs, or security issues;
- translating documentation;
- summarizing issues or pull requests.

## Restricted Uses of AI

Do not use AI tools to:

- intentionally reproduce code from another project without checking license compatibility;
- bypass copyright, license, attribution, or contributor obligations;
- generate security-sensitive code without human review;
- generate cryptographic, authentication, authorization, payment, privacy, or data-deletion logic without additional scrutiny;
- submit code you do not understand;
- flood the project with low-effort generated pull requests;
- create fake benchmarks, fake tests, fake citations, or misleading documentation;
- include private, confidential, personal, customer, or proprietary data in prompts unless you are authorized to do so;
- submit AI output that you cannot explain, test, and maintain.

## Disclosure

Small AI-assisted edits, autocomplete suggestions, formatting changes, and documentation rewrites do not require special disclosure.

Material AI assistance should be disclosed in the pull request when AI was used to create a substantial part of the design, implementation, tests, or documentation.

Examples of material AI assistance include:

- generating a new module or major function;
- generating a large test suite;
- translating an implementation from another language;
- porting an algorithm;
- drafting security-sensitive logic;
- producing complex SQL, data processing, geospatial, entity-resolution, or matching logic.

A simple disclosure is sufficient:

> This PR was AI-assisted. I used an AI tool to draft the initial implementation/tests/docs. I reviewed, modified, tested, and accept responsibility for the contribution.

## License and Provenance

Contributors must ensure that their contributions are compatible with this project's license.

If an AI tool produces output that appears copied from, derived from, or substantially similar to another project, the contributor must:

- identify the source if known;
- check the source license;
- ensure compatibility with this project's license;
- add attribution if required;
- rewrite the contribution if compatibility is unclear.

When in doubt, do not submit the AI-generated code.

## Security and Safety

AI-assisted code must meet the same security standard as human-written code.

## Testing Requirements

AI-assisted contributions must include appropriate tests.

For behavior-changing code, contributors should include tests for:

- expected behavior;
- edge cases;
- failure cases;
- malformed input;
- ambiguous input;
- regression cases;
- security-sensitive cases where applicable.

Generated tests should be reviewed carefully. Tests must verify the intended behavior, not merely reproduce the assumptions of generated code.

## Maintainer Rights

Maintainers may ask contributors to:

- disclose whether AI materially assisted the contribution;
- explain the implementation;
- provide additional tests;
- simplify or rewrite generated code;
- provide provenance information;
- remove code with unclear licensing;
- split large AI-generated pull requests into smaller reviewable changes.

Maintainers may reject AI-assisted contributions that are difficult to review, insufficiently tested, legally unclear, insecure, misleading, or inconsistent with the project's design.

## Contributor Certification

By submitting a contribution, you certify that:

1. You have reviewed and understand the contribution.
2. You have the right to submit it under this project's license.
3. You are not knowingly submitting incompatible licensed code.
4. You have not included confidential or private data without authorization.
5. You have tested the contribution appropriately.
6. You accept responsibility for fixing issues introduced by the contribution.
